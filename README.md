# SiPush — Obsidian ↔ 思源笔记 双向同步插件

**V2.0.0** — 纯 JS 编写，零构建，直接放入插件目录即可使用。

## 功能

| 功能 | 说明 |
|------|------|
| **推送** | 当前笔记 → 思源（已有文档原地更新，永不重复） |
| **双向同步** | 对比时间戳 + 内容哈希，自动决定 push 或 pull |
| **批量同步** | 遍历所有关联笔记，逐个双向同步并展示报告 |
| **搜索拉回** | 从思源搜索已关联文档，拉取到 Obsidian |
| **冲突处理** | 双方都修改时弹窗让用户选择保留哪边 |

## 安装

1. 将本目录放入 Obsidian 插件目录：
   `.obsidian/plugins/obsidian-si-push/`
2. Obsidian → 设置 → 第三方插件 → 启用 SiPush
3. 在插件设置中配置：
   - 思源服务器地址（默认 `http://127.0.0.1:6806`）
   - API Token（思源设置 → 关于 → API Token）
   - 默认笔记本（点击「刷新列表」获取）

## 使用

### 命令面板（Ctrl/Cmd+P）

| 命令 | 说明 |
|------|------|
| `推送当前笔记到思源` | 将当前打开的笔记推送到思源 |
| `推送选中内容到思源` | 推送选中的文本到思源 |
| `追加到思源已有文档` | 搜索思源文档并追加内容 |
| `双向同步当前笔记` | 对比 Obsidian 和思源，自动同步 |
| `批量双向同步所有关联笔记` | 遍历所有已关联笔记进行双向同步 |
| `搜索思源文档并拉回` | 从思源搜索并拉取到 Obsidian |
| `测试思源连接` | 测试连接是否可用 |

### 关联机制

首次推送时，插件会在笔记 frontmatter 中自动写入：

```yaml
---
custom-si-push-id: si1y7z8a3b2c1d
---
```

此 ID 同时写入思源文档的属性中，作为双向关联的唯一标识。

### 同步策略

1. **内容相同** → 不操作
2. **Obsidian 更新** → 推送到思源
3. **思源更新** → 拉取到 Obsidian
4. **同时修改** → 按冲突解决策略处理

冲突解决策略可在设置中选择：
- **弹窗让用户选择**（默认）
- **始终保留 Obsidian 版本**
- **始终保留思源版本**

## 思源文档结构

```
/Obsidian/                ← 默认路径前缀
├── 笔记标题_A.md          ← 新推送的文档
└── 笔记标题_B.md
```

每个文档带有以下属性：
- `custom-si-push-id` — 关联 ID
- `si-push-content-hash` — 内容哈希
- `si-push-mtime` — 最后修改时间戳
